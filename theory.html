Fast Gradient Sign Method (FGSM) Formulation

Maximizing classifier loss using L-infinity sign-noise directions

Core Mathematics

Introduced by Goodfellow et al. (2014), the Fast Gradient Sign Method (FGSM) is a simple, highly efficient one-step adversarial attack. Instead of minimizing classification loss (as we do during training to improve accuracy), FGSM performs gradient ascent to maximize model classification loss with respect to the input pixels.

By taking a step in the direction of the sign of the loss gradients, the attack ensures that every pixel value is shifted by a maximum step of $\epsilon$, maximizing L-infinity perturbation boundaries:

x_{adv} = x + \varepsilon \cdot \text{sign}(\nabla_x \mathcal{L}(\theta, x, y))
x_adv : Perturbed Adversarial Image
x : Original Clean Input Image
ε (Epsilon) : Perturbation size bound
∇_x L(θ, x, y) : Loss gradient w.r.t input pixels

Adversarial Optimization Pipeline

Step-by-step workflow of gradient-based pixel perturbation

1
Forward Propagation & Loss Calculation Input $x$ is passed through the network weights $\theta$ to calculate classification cross-entropy loss $L(\theta, x, y)$ with respect to true label $y$.
2
Backward Pass to Input Pixels Backpropagation is performed. Instead of calculating gradients w.r.t model weights, we compute the gradient of the loss specifically w.r.t each input pixel: $\nabla_x L$.
3
Sign Extraction & Scaling We extract the sign of the calculated gradients ($\pm 1$). Multiplying by $\epsilon$ creates the optimal perturbation mask bounded by the L-infinity norm.
4
Perturbation Injection & Pixel Clamping The scaled mask is added to the original image. Pixels are clamped to $[0, 1]$ to remain valid, resulting in the completed adversarial image $x_{adv}$.

Projected Gradient Descent (PGD)

Multi-step iterative sign attack

Iterative Attack

PGD is an iterative extension of FGSM. It takes multiple smaller gradient sign steps (step size $\alpha$) and projects the resulting image back onto the $\epsilon$-ball (the search domain $\mathcal{S}$) surrounding the original image at each step. This acts as a highly potent optimization attack:

x^{t+1} = \Pi_{x + \mathcal{S}} \left( x^t + \alpha \cdot \text{sign}(\nabla_x \mathcal{L}(\theta, x^t, y)) \right)

Note: PGD is considered the ultimate L-infinity first-order attack. A model robust to PGD attacks is generally robust to all other L-infinity bounded first-order attacks.

def fgsm_attack(image, epsilon, data_grad):
    # 1. Collect the sign of the input gradients
    sign_data_grad = data_grad.sign()
    
    # 2. Add the perturbation to the original image pixels
    perturbed_image = image + epsilon * sign_data_grad
    
    # 3. Clip the perturbed image pixels to maintain valid normalized range [0, 1]
    perturbed_image = torch.clamp(perturbed_image, 0, 1)
    
    return perturbed_image

def pgd_attack(model, image, target, epsilon, alpha=0.01, num_iter=10):
    original_image = image.clone().detach()
    perturbed_image = image.clone().detach()
    perturbed_image.requires_grad = True
    loss_fn = nn.CrossEntropyLoss()
    
    for _ in range(num_iter):
        output = model(perturbed_image)
        loss = loss_fn(output, target)
        model.zero_grad()
        loss.backward()
        
        data_grad = perturbed_image.grad.data
        
        with torch.no_grad():
            perturbed_image = perturbed_image + alpha * data_grad.sign()
            # Projection step: constrain perturbation to L-infinity norm ball
            eta = torch.clamp(perturbed_image - original_image, min=-epsilon, max=epsilon)
            perturbed_image = torch.clamp(original_image + eta, min=0, max=1).detach()
            
        perturbed_image.requires_grad = True
        
    return perturbed_image.detach()
