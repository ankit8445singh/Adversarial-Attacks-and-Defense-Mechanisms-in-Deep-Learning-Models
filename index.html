Problem Definition & Objectives

Understanding security vulnerability in Applied Machine Learning

AML Project Core

Deep learning models have achieved remarkable, human-level accuracy in tasks such as image classification, but they remain highly vulnerable to adversarial attacks—carefully crafted pixel perturbations added to input data that can mislead models into making confident, incorrect predictions. These perturbations are often completely imperceptible to humans, yet they catastrophically degrade model performance.

This vulnerability raises serious security and safety concerns in real-world applications such as autonomous driving, healthcare diagnostic networks, and biometric validation systems, where incorrect classification predictions can lead to critical real-world consequences.

Primary Objective: To analyze how adversarial perturbations affect deep learning classifiers and to implement robust defense mechanisms (like Adversarial Training and Input Preprocessing) to restore model stability and reliability.

Project Key Objectives Checklist:

Implement FGSM & PGD Attacks
Evaluate Epsilon Decay Slopes
Design Input Preprocessors
Construct Robust Adv Training
